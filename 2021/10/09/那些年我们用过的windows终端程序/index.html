那些年我们用过的windows终端程序

发表于 2021-10-09 分类于 花里胡哨的东西 阅读次数： Valine：
本文字数： 2.3k 阅读时长 ≈ 5 分钟

自从安装完windows11，我想我的windows终端折腾之旅也算告一段落，回顾曾经折腾过的这么多终端模拟器，简单总结一下自己的退烧之路吧。

1.Cmder

Cmder初见真是惊为天人呀！那时的我还没有linux服务器可以玩，也不懂租服务器啥的，只有一个破破的windows笔记本可以跑实验。cmder可以说是陪伴我大学最后两年的最强终端！我后面的所有美化也基于它来做的，那就谈谈他的优点吧。

  • 自带许多shell命令。如head, cat, wc, mv,cp，curl，vim，sh等基础命令，方便简单查看数据，以及进行简单的编辑，甚至还能支持一些shell脚本的编写和运行，跑实验很舒服有没有！
  • 自带git并做了提示符美化。可以在提示符里清晰的查看本地分支与关联的远程分支，同时可以通过颜色来提示改动，另外也省去了我再下载一个git。嗯，我重装系统一般都是只去下个cmder。
  • 支持alias。这个可以说是帮了大忙了，我巨巨巨巨不喜欢那种又长又恶心的命令，我喜欢简单的，所以我设置了很多命令的别名，用起来效率自然是大大提高！
  • 支持一些快捷键。自带history文件，可以Ctrl+r进行历史命令搜索，还有一些Ctrl+e, Ctrl+b，Ctrl+w等，方便了命令的输入和修改。
  • 比较美观，具备一定的可定制性。比系统自带的powershell和cmd不知道高到哪里去了，而且还支持非常多的设置，比如支持设置启动脚本，我想启动cmder的时候直接连接实验室的服务器。只需要在启动脚本里加上ssh xxxxxxxx就行了。
  • 支持右键菜单栏。这个可能每个终端都支持，但是cmder只需要一条命令就完成，省去了写ps脚本的过程。配置完成之后，我开终端就可以通过文件浏览器去定位到文件，然后鼠标右键，直接选择通过cmder打开就可以啦~
  • 可以作为VS Code的默认终端！你的所有美化都将被继承！如果你正在使用VS Code，那这绝对是一款神器！

当然，cmder虽然这么好，但是它的缺点是什么呢？

一个字 丑！

我曾经尝试为其更改了许多主题，字体，还定制了一些提示符，当然我也为此写了一篇博客Cmder高亮以及修改背景。但是美化程度终究有限，为了做进一步的美化，我接触了如下的几款终端美化程序，然后为这些程序配置了一个启动时自动运行cmder的脚本

2.Fluent Terminal+Cmder

Fluent Terminal的优点自然就是好看，我看中它的自然也是好看，因为它有好看的字体，主题，以及背景的毛玻璃效果。

那他的缺点是什么呢？

  • 启动慢。我将其添加到右键菜单后，通过右键启动的速度非常的慢！
  • 有bug。比如闪退bug，比如ssh连到服务器后tmux乱码的bug。
  • 与cmder兼容有限。我在cmder中配置的东西，在这里会有部分失效，这让我十分无法接受。
  • 美化困难。可配置的内容有限，而且资料不太好找。

3.Windows Terminal+Cmder

后来我就馋上了巨硬家的windows terminal，其实乍一看这个terminal和fluent terminal差别不大，其实确实如此，他们真的在外观上没啥差别！那么它的优点在哪呢？

  • 大牌子。哈哈哈，这个自然是优点，windows自家做的，而且在新来的windows11中，这个已经是默认的终端程序了。大牌子就意味着可能稳定性好，而且会持续维护。
  • 与cmder的完美兼容！这一点实在是太棒了，不会和我在cmder的配置产生冲突！
  • 可定制化强。windows terminal和vscode 差不多，都可以通过编辑json文件为其配置主题，字体等，而且其有专门的主题网站Windows Terminal Themes，可以在这个网站上实时预览你的终端效果，选择困难症表示我可以挑一下下午…

4.Windows Terminal + WSL2

这两天更新了正式版的windows 11，我心心念念的wsl2终于可以安装cuda了 。其实去年似乎就可以了，但是我当时更新了一次，发现系统有bug就回退了。那么现在我就可以真真正正的在windows中使用linux跑程序了，而且据说这波GPU的性能损失很低？附一张pytorch成功运行cuda的图~

那么为什么cmder没有wsl2香？

  • 首先，wsl2可以很自然的访问windows文件。虽然它是一个linux子系统，但是windows的文件，它都可以访问的到（硬盘被挂载到了/mnt下），这意味着我可以使用windows写程序，然后用linux跑~
  • 使用wsl2，我就可以使用oh-my-zsh。oh-my-zsh的美化是非常成熟的。而且最香的是一些插件，比如auto-suggestion，zsh-syntax-highlighting。当然由于是linux所以还有tmux，ranger等插件的使用。这些插件在windows平台都没有，而这些都可以极大的提高生产力。
  • wsl2的shell命令更加丰富。cmder是模拟的linux，所以支持的命令有限，比如htop,watch等就不支持，而wsl2是真正的linux，自然是随便用啦~
  • 迁移服务器配置！哈哈哈哈，我花了大心血给服务器配置的一套使用方案，可以无缝迁移到windows啦，这当然是最开心的啦~

其实还有一些终端也有用过，但时候不长，这里就不提了。现在想想，玩了这么多花里胡哨的东西，也该退烧了~毕竟自己的科研任务还没有起色，也不能一拖再拖了~ 这波就当给大家安利windows terminal以及wsl2了。当然cmder还是windows下永远的神！
